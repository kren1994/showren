# JSONフォーマット

保存ファイルは UTF-8 の JSON です。トップレベルは以下の5フィールドです。

```json
{
  "t": { ... },
  "p": { ... },
  "c": "現在のノードID",
  "n": 次のノード番号,
  "pl": { ... }
}
```

| フィールド | 説明 |
|---|---|
| `t` | 棋譜ツリー（ノードIDをキーとするオブジェクト） |
| `p` | 局面データベース（局面キーをキーとするオブジェクト） |
| `c` | 現在表示中のノードID |
| `n` | 次に採番するノード番号 |
| `pl` | 局面ラベル辞書。`byPosKey` と `byName` を持つ |

---

### ツリーノード (`t` の値)

各ノードは1手を表します。ルートノードのIDは `"r"` です。

```json
"r": {
  "i": "r",
  "p": null,
  "c": ["1", "2"],
  "x": null,
  "y": null,
  "o": 0,
  "m": 0,
  "l": "1"
}
```

| フィールド | 説明 |
|---|---|
| `i` | このノード自身のID（ルートは `"r"`、それ以外は採番された整数文字列） |
| `p` | 親ノードのID（ルートは `null`） |
| `c` | 子ノードIDの配列 |
| `x` | 交点の列（0〜14、ルートは `null`） |
| `y` | 交点の行（0〜14、ルートは `null`） |
| `o` | 石の色（`1`=黒、`2`=白、`0`=ルート） |
| `m` | 手数（ルートは `0`） |
| `l` | 最後に選択した子ノードのID（省略可） |

---

### 局面データベース (`p` の値)

局面の対称性を考慮したカノニカルキーをキーとします。

```json
"ABcd...xyz1": {
  "c": "ここに検討コメント",
  "l": { "7,7": "A", "8,7": "B" },
  "n": { "7,7": 1, "8,7": 1 }
}
```

**キーの形式：** 盤面225マス（15×15）の石配置を 5トリット/バイト で45バイトに圧縮し、base64url エンコードした60文字＋手番色1文字の計61文字。

| フィールド | 説明 |
|---|---|
| `c` | 局面コメント（文字列） |
| `l` | 交点ラベル。キーは `"x,y"` 形式（カノニカル座標）、値は表示テキスト |
| `n` | 次の手の登録済み交点。キーは `"x,y"` 形式、値は `1` |

> `l` と `n` の座標はカノニカル座標（対称変換後）で記録されます。盤面上の物理座標とは向きが異なる場合があります。

---

### 局面ラベル / 索引 (`pl` の値)

`pl` は任意キーですが、保存時は常に出力されます。旧形式の `pl` なし JSON も読み込み可能です。

```json
"pl": {
  "byPosKey": {
    "ABcd...xyz1": {
      "k1": "2026",
      "k2": "関東選手権",
      "k3": "1",
      "rep": "37"
    }
  },
  "byName": {
    "2026/関東選手権/1": "ABcd...xyz1"
  }
}
```

| フィールド | 説明 |
|---|---|
| `byPosKey` | `posKey -> { k1, k2, k3, rep }` の辞書 |
| `byName` | `k1/k2/k3 -> posKey` の辞書 |

`rep` はそのラベルを登録した時点の代表手順ノード ID です。ジャンプ時はまずこのノードを使い、存在しない場合のみ同一局面の探索にフォールバックします。

UI 上ではこの機能は右ペインの `索引` タブとして表示されます。
